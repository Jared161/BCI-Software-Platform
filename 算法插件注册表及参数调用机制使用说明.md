# 算法插件机制使用说明
## 一、新算法添加规范
### 核心要求
1. 新增算法必须继承 `BaseAlgorithm` 基类，实现 `train()`、`predict()` 抽象方法，以及 `name` 特征属性（返回唯一算法名称）；
2. 算法类定义前需添加 `@register_algorithm` 装饰器，用来实现自动注册；
3. 算法文件直接放入 `algorithms` 目录，注册表会自动发现加载，无需修改主程序/注册表代码。
综上，新增代码必须有类似于以下几行代码：

from ..base import BaseAlgorithm
from .. import register_algorithm
@register_algorithm
'''

'''

## 二、输出指标管理规则
### 现有逻辑
1. 算法模块需要有计算逻辑和接受主程序所传递数据的相应变量格式，最好有输出逻辑
2. 主程序仅保留「算法注册→实例化→调用」的核心逻辑，无需为每个算法编写分支。
   另外，主程序会读取加载数据集文件，并将相应数据传递给将要运行的算法。
   换句话说，后续主程序可增加根据提供的数据集类型选择算法的功能。

## 三、理想效果
- 算法模块包含数据处理、计算逻辑以及输出，完全独立，即插即用，可单独测试或迭代。
- 主程序只负责注册、实例化以及调用算法，无算法专属逻辑，降低多人协作冲突；
